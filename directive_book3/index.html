This example shows how isolate scope binding works.

  • Use @ when you want to pass one way string
  • Use = when you want to have two way variable
  • Use & when you want to pass ctrl function to be invoked

    • Note: With @ you are passing string. Whatever you specified in directive as attribute
    that will be converted into a string and passed to the directive. (see textVar)
    Directive can use it as own variable. If it is udated in Ctr change will be propagated to the directive
    However, if you change it in the directive, it will not be propagated to the Ctr. (see ChangeDirVars button)
    With = you are telling directive to use attribute's value to find out property on ctrl scope
    that binds to this directive scope variable. (see myTwoWay). Here directive attribute value is: twoWayVar
    So, whenever you refer to rwVar, you are actually referring to Ctrl: $scope.twoWayVar
    With &, you are telling directive to use attribute value as ctrl function to be invoked every time
    this local function gets invoked. (see action)
